🔧 Instrukcja Testowania - Plan Synchronization

🚀 Szybkie linki do testowania

📋 Krok 1: Przygotowanie do testów

1. Otwórz konsolę deweloperską (F12 w przeglądarce)

2. Przejdź na zakładkę Console

3. Wyczyść konsolę (Ctrl+L lub przycisk Clear)

Teraz wszystkie logi będą widoczne w konsoli deweloperskiej

🧪 Krok 2: Test z Symulatorem Problemu

1. Otwórz problem-simulator.html

2. Kliknij "Ustaw Premium" - sprawdź czy wszystko się synchronizuje

3. Kliknij "🚨 SYMULUJ PROBLEM" - to odtworzy sytuację z różnymi planami

4. Obserwuj logi - czy pokazuje "PROBLEM: SessionStorage ma Enterprise, ale userPlan pokazuje Premium"

5. Kliknij "🔄 Wymuś synchronizację" - sprawdź czy to naprawia problem

🎯 Krok 3: Test na głównej stronie

1. Otwórz baza_cv_new.html

2. W konsoli ustaw dane użytkownika:

sessionStorage.setItem('currentUser', JSON.stringify({ name: 'Test User', email: 'test@example.com', plan: 'Premium' }));

3. Odśwież stronę (F5)

4. Obserwuj logi - szczególnie zwróć uwagę na:

  • Czy ultimatePlanSync() jest wywoływana wielokrotnie
  • Czy są jakieś niespodziewane zmiany w DOM
  • Czy sessionStorage się zmienia

🔍 Krok 4: Debugowanie w czasie rzeczywistym

W konsoli dostępne są funkcje:

// Pokaż aktualny stan debugUserState() // Wymuś synchronizację forcePlanSync() // Zmień plan testowo testPlanChange('Enterprise')

⚠️ Na co zwrócić uwagę

Szukaj w logach:

  • 🔴 "NIEZGODNOŚĆ PLANU" - to oznacza problem!
  • 🔄 Wielokrotne wywołania ultimatePlanSync()
  • 🎯 Call stack - skąd pochodzi wywołanie
  • 📋 Zmiany DOM - czy ktoś modyfikuje elementy bezpośrednio

🚨 Krok 5: Reprodukcja problemu z życia

Aby odtworzyć dokładnie problem "dalej w jednym miejscu jest premium a w drugim enterprise":

// 1. Ustaw Premium sessionStorage.setItem('currentUser', JSON.stringify({ name: 'Test User', email: 'test@example.com', plan: 'Premium' })); // 2. Poczekaj 2 sekundy, potem zmień na Enterprise setTimeout(() => { sessionStorage.setItem('currentUser', JSON.stringify({ name: 'Test User', email: 'test@example.com', plan: 'Enterprise' })); }, 2000);

Wykonaj ten kod w konsoli i obserwuj czy elementy się prawidłowo synchronizują.

✅ Oczekiwane rezultaty

Jeśli wszystko działa poprawnie:

  • Elementy userWelcome i userPlan pokazują te same dane
  • Zmiany w sessionStorage od razu aktualizują wyświetlanie
  • ultimatePlanSync() wykonuje się bez błędów

Jeśli występuje problem:

  • userWelcome pokazuje jeden plan, userPlan inny
  • Logi pokazują "NIEZGODNOŚĆ PLANU"
  • Call stack pomoże znaleźć źródło problemu

🔧 Jak naprawić wykryty problem

Jeśli znajdziesz problem w logach:

  1. Sprawdź call stack - skąd pochodzi niewłaściwe wywołanie
  2. Szukaj funkcji która modyfikuje DOM lub sessionStorage
  3. Dodaj console.log w podejrzanych miejscach
  4. Wyłącz czasowo automatyczne odświeżanie (setInterval)
